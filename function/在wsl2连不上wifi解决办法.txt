当时是用cmd进入wsl2，然后更改wslconfig文件能容，然后用powershell管理者模式下退出wsl --shutdown，再进入wsl。我就改了wslconfig文件，但是后面的sudo nano /etc/wsl.conf里面的内容好像是因为设置了.wslconfig后自动生成的：
[network]
# 以下示例配置自动获取 IPv4 地址和默认网关
generateResolvConf = true
generateHosts = true
generateInterfaces = true
interfaceName = eth0
ipAddress = dhcp
netmask = dhcp
gateway = dhcp
--------------------------------------------------------------------------
